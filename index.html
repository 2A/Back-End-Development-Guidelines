Code Layout

When indenting code, use tabs, not spaces!

Define arrays / multi-dimensional arrays on multiple lines.

Define functions with bracet on new line.

No spacing / indenting when doing multi-dimensional arrays (for example). It becomes a waste of time trying to indent everything, and then when your wife wants to change a word, you have re-indent everything!

When you have a bunch of methods to chain, it becomes much more legible if you put them on new lines. But if there are only a few methods to chain, odly enough, it is easier to read them on a single line.

Defining Methods, Variables, Constants

With much discussion, and fighting, and brawling, and some recovery programs, we decided to go for lowercase method names and variable names. Some of us just don't want to give up python standards.

Loose Ends: Coding Preferences

When using bool, null, constants, etc. The uppercase usage is preferred.

Conditionals for TRUE, FALSE, NULL, etc read, "If var is true, then..." Not "If true is var". Even though TRUE == $var is safer, it is agreed that we would rather debug it than write it that way.

Add a space between the ! and variable. Not before it, not after the variable.
